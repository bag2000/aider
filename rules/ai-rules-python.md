# Python Rules для сисадмина

## Назначение
- Автоматизация, утилиты, админские скрипты
- Не application development
- Python 3.9+

## Стиль
- Процедурный код, функции > классов
- Читаемость важнее всего
- Один файл = одна задача
- `if __name__ == "__main__":` + `main()`

## Практика
- Стандартная библиотека в приоритете
- Не логировать секреты
- Использовать `pathlib` для путей
- Обработка ошибок: понятные сообщения, ловить только ожидаемое

## Структура
- Использовать `logging` вместо `print`
- Не хардкодить пути/порты/креды
- Докстринги для скриптов и нетривиальных функций
- Комментарии объясняют "почему"
