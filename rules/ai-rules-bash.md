# Bash Rules для сисадмина

## Основное
- Bash только для системной автоматизации, не для сложной логики
- Используй `#!/usr/bin/env bash` и Bash 4+
- По умолчанию: `set -Eeuo pipefail`

## Безопасность
- Всегда `"$var"` с кавычками
- Никогда `rm -rf "$var"` без проверки
- Не используй `eval`
- Проверяй права и владельцев для файлов с секретами

## Структура
- Один скрипт = одна задача
- `UPPER_CASE` для констант, `lower_case` для переменных
- Функции для логических блоков
- Проверяй наличие зависимостей (`command -v`)

## Практика
- Используй `[[ ]]` вместо `[ ]`
- `mktemp` для временных файлов + `trap` для очистки
- Проверяй `$UID` если нужен root
- Логирование: `log_info`, `log_error` (в stderr)

## Вывод
- Поддерживай `-h/--help`
- Объясняй риски в опасных местах
- Комментарии объясняют "зачем", а не "что"
